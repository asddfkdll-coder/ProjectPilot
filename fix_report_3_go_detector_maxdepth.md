# تقرير الإصلاح: الكشف عن `main.go` في `GoProjectDetector`

## اسم المشكلة

الكشف عن `main.go` في `GoProjectDetector` قد يفشل في اكتشاف مشاريع Go التي تحتوي على ملف `main.go` في مجلد فرعي أعمق.

## مكانها

`app/src/main/java/com/projectpilot/app/data/scanner/Detectors.kt`، السطر 172.

## سببها

كان `GoProjectDetector` يبحث عن ملف `main.go` بحد أقصى للعمق يبلغ 2 (`maxDepth(2)`). هذا قد لا يكون كافيًا لمشاريع Go ذات البنية الأكثر تعقيدًا حيث قد يكون ملف `main.go` موجودًا في مجلدات فرعية أعمق.

## الكود القديم

```kotlin
        val main = dir.walkTopDown().maxDepth(2).firstOrNull { it.name == "main.go" }
```

## الكود الجديد

```kotlin
        val main = dir.walkTopDown().maxDepth(5).firstOrNull { it.name == "main.go" } // Increased depth for better detection
```

## سبب اختيار الحل

تم زيادة `maxDepth` من 2 إلى 5. هذا يسمح للكاشف بالبحث في مجلدات فرعية أعمق، مما يزيد من احتمالية اكتشاف ملف `main.go` في مشاريع Go ذات البنية المتنوعة. تم اختيار القيمة 5 كحل وسط بين البحث الشامل وتجنب استهلاك الموارد المفرط.

## هل يوجد تأثير جانبي؟

قد يؤدي زيادة عمق البحث إلى استهلاك المزيد من الموارد (وقت المعالجة والذاكرة) عند مسح المشاريع الكبيرة جدًا. ومع ذلك، بالنسبة لمعظم مشاريع Go، يجب أن يكون التأثير ضئيلًا.

## هل يحتاج اختبار إضافي؟

نعم، يحتاج إلى اختبار إضافي مع مشاريع Go ذات هياكل مجلدات مختلفة للتأكد من أن الكاشف يعمل بشكل صحيح ولا يزال فعالًا من حيث الأداء.
